UDP和TCP的对比：
	 
    UDP和TCP都是传输层的协议。其中
            UDP是一种面向无连接，不可靠的传输协议。

            面向无连接指的的发送数据之前不会提前建立连接来确定是否能和对方主机进行通信
            不可靠体现在数据包传输过程中 丢包不会重发，包顺序混乱也不会做处理，UDP就是
            尽我最大可能的传输数据。

            但UDP有其适合的场景：如即时通讯，视频会议之类的
			--------------------------------------------------
			TCP是一种面向有连接的，可靠的传输协议。
			面向有连接指的是：
				创建连接时先会进行3次握手来确定能和对方主机进行通信后在创建数据连接
				断开连接时会进行4次挥手来保证连接能够正常断开
				
			可靠性体现在没收到数据包，会进行重发，根据数据包的序号和应答序号可以保证数据
			是有序传输。
            此外TCP还可以通过窗口控制来提高传输效率，以及流量控制来保证接收端
			能及时并合理处理发送过来的数据，拥塞控制来避免大数据量传出对网络造成过重的负载
           
        	TCP比较适合需要可靠传输的业务场景，如web服务
			
   三次握手
	    1、客户端向服务端发 SYN 建立连接的请求，
    	2、服务端收到后确认应答ACK并向客户端发送一个SYN建立连接的请求，
        3、客户端收到后确认应答ACK，正式创建连接
   为什么是三次握手而不是两次握手？
		首先 TCP 协议是一种可靠性传输，它的可靠性体现在保证数据的完整性和有序性
    	如果仅仅是两次握手的话，那就只能保证从客户端发送过来的数据是完整的有序的，但
        从服务端发送给客户端的数据不一定是有序的。因为缺少第二次握手的话，服务端就没法
        告知客户端我的初始序列号是多少，如果你要给我发数据从哪里发起。	
   
   四次挥手
   		1、客户端向服务端发送 FIN的断开连接的请求
    	2、服务端收到后确认应答ACK 【TCP连接已经处于一个半闭合的状态】
        3、服务器端向客户端发送FIN断开连接的请求
        4、客户端收到后去应答ACK断开连接
   四次挥手的原因是保证服务端能完成的传输数据

socket是把 tcp/ip 复杂的操作抽象成可编程的接口来供应用层调用		

    # 文件名：server.py

    import socket               # 导入 socket 模块

    s = socket.socket()         # 创建 socket 对象
    host = socket.gethostname() # 获取本地主机名
    port = 12345                # 设置端口
    s.bind((host, port))        # 绑定端口

    s.listen(5)                 # 等待客户端连接
    while True:
        c,addr = s.accept()     # 建立客户端连接
        print '连接地址：', addr
        c.send('欢迎访问菜鸟教程！')
        c.close()                # 关闭连接


    # 文件名：client.py

    import socket               # 导入 socket 模块

    s = socket.socket()         # 创建 socket 对象
    host = socket.gethostname() # 获取本地主机名
    port = 12345                # 设置端口号

    s.connect((host, port))
    print s.recv(1024)
    s.close()
